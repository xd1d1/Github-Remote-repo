git init // um den status in git einzusehen.
git add <file>
git commit -m "your commit message"
git add . **gibt den ganzen Ordner mit.
git push -u origin main // um von git commit nach github res

Wenn alles noch nicht auf github ist
git reset --soft HEAD~1 // nimmt nur den letzetn commit , aber die Önderungen werden in staging wieder zurück gepackt
git reset --mixed HEAD~1 // löscht den letzten commit komplett raus, und wird ins working directory gepackt (also arbeitsbereich)
git reset --hard HEAD~1 // löscht alles komplett weg. Also kommt weder ins staging rein, noch weder ins working directory
